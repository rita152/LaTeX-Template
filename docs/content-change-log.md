# slide.tex 内容修改汇总

> 本文档用于持续记录 `slide.tex` 的结构与内容变更，方便在后续组会中快速回溯调整思路。每次修改后请新增一条变更记录，并同步更新维护建议。

## 文档定位
- 记录：说明此次修改的动机、具体调整点与影响范围。
- 指南：提供下一次修改需要关注的注意事项、引用规范与编译方式。
- 追踪：保持与 `pic/`、`XMU.sty` 等依赖文件的同步状态，避免资源缺失。

## 变更日志

- **执行概览**
    - 完成 `slide.tex` 由模板示例到“多模态行为识别组会汇报”的整体替换。
    - 封面使用“张鹏 · 厦门大学（XMU）航空航天学院 · 当日日期”，并引用 `pic/xmu-logo.pdf`。
    - 通过 `./build_slide.sh` 验证可编译，脚本同时清理 `.bbl` 等中间文件。
- **主要调整内容**
    1. \textbf{封面与元信息}：统一 XMU 身份信息，副标题直接展示当日日期，Logo 换成 `pic/xmu-logo.pdf`。
    2. \textbf{章节重构}：章节顺序更新为“课题背景与目标 → 数据与方法 → 实验设计与结果 → 问题与讨论 → 下一步计划 → 参考文献”。
    3. \textbf{内容更新}：替换为项目背景、数据采集表、系统方案、实验设置/性能表、讨论要点、里程碑与行动项等真实内容。
    4. \textbf{视觉统一}：保留 `XMU.sty`，但所有描述聚焦项目叙事；箭头符号改用 `$\rightarrow$` 以避免字体缺失警告。
    5. \textbf{参考文献}：沿用 `\bibliography{ref}` + `xmu.bst`，当前未添加 `\cite{}`，BibTeX 会提示“no \citation”，属预期行为。
- **当前章节结构**
    1. `Section 1`：课题背景与目标（项目背景、研究目标与指标）
    2. `Section 2`：数据与方法（数据采集表、系统方案概览）
    3. `Section 3`：实验设计与结果（实验设置、性能对比、可视化案例）
    4. `Section 4`：问题与讨论（挑战与风险、讨论要点）
    5. `Section 5`：下一步计划（里程碑安排、本周行动项）
    6. `Section 6`：参考文献（沿用 `ref.bib`）

## 维护指引
- **引用管理**：添加参考文献时务必在正文处补充 `\cite{}`，再运行 `./build_slide.sh` 以生成新的 `.bbl`；脚本会自动清理 `.bbl`，如需保留请手动备份。若引用风格需调整，请直接修改 `xmu.bst`。
- **素材规范**：新增图示统一放入 `pic/`，优先使用矢量格式（PDF/EPS）；引用路径示例 `\includegraphics{pic/xmu-logo.pdf}`。
- **主题一致性**：如需调整配色或页脚样式，集中在 `XMU.sty` 修改，避免在 `slide.tex` 内硬编码。
- **编译流程**：默认使用 `latexmk -xelatex`，遇到符号缺失可优先检查是否使用了非 ASCII 字符或缺失字体。
- **记录更新**：每次改动 `slide.tex` 后，请在本文件新增日期条目并描述变更摘要，以便团队成员同步。
